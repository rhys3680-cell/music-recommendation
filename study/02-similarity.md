# 2. 유사도 측정

추천 시스템의 핵심은 **"비슷한 것"을 찾는 것**이다. 비슷한 사용자, 비슷한 아이템을 찾으려면 "비슷하다"를 수학적으로 정의해야 한다.

## 왜 유사도가 중요한가?

```
협업 필터링:  "나와 비슷한 사용자가 좋아한 것을 추천" → 비슷한 사용자를 어떻게 찾는가?
콘텐츠 기반:  "이 아이템과 비슷한 아이템을 추천" → 비슷한 아이템을 어떻게 찾는가?

→ 유사도 함수 sim(A, B)가 필요하다
```

## 주요 유사도 측정 방법

### 1. 코사인 유사도 (Cosine Similarity)

두 벡터 사이의 **각도**를 기반으로 유사도를 측정한다. 벡터의 크기는 무시하고 **방향만 비교**한다.

#### 수식

```
              A · B           Σ(Ai × Bi)
sim(A,B) = --------- = ----------------------
            |A||B|      √Σ(Ai²) × √Σ(Bi²)
```

- 결과 범위: -1 ~ 1 (양수 값만 다루면 0 ~ 1)
- 1에 가까울수록 유사, 0이면 무관, -1이면 반대

#### 예시

```
유저A = [5, 3, 0, 1]   (영화 4편에 대한 평점)
유저B = [4, 0, 0, 5]

A · B = (5×4) + (3×0) + (0×0) + (1×5) = 25
|A| = √(25 + 9 + 0 + 1) = √35 ≈ 5.92
|B| = √(16 + 0 + 0 + 25) = √41 ≈ 6.40

sim(A,B) = 25 / (5.92 × 6.40) ≈ 0.66
```

#### 특징

- **스케일에 무관**: 유저A가 [5,3,0,1], 유저C가 [10,6,0,2]이면 sim = 1.0 (같은 방향)
- 후한 평점을 주는 사용자와 짠 사용자를 동일하게 취급
- 추천 시스템에서 **가장 널리 사용**되는 유사도

---

### 2. 피어슨 상관계수 (Pearson Correlation)

각 사용자의 **평균 평점을 빼고** 유사도를 계산한다. 즉, 평점의 절대값이 아니라 **상대적 경향**을 비교한다.

#### 수식

```
             Σ((Ai - Ā)(Bi - B̄))
sim(A,B) = --------------------------
            √Σ(Ai - Ā)² × √Σ(Bi - B̄)²
```

- Ā = 유저A의 평균 평점
- 결과 범위: -1 ~ 1

#### 예시

```
유저A = [5, 3, 4]    평균 Ā = 4.0
유저B = [3, 1, 2]    평균 B̄ = 2.0

보정 후:
유저A' = [1, -1, 0]
유저B' = [1, -1, 0]

sim(A,B) = (1×1 + (-1)×(-1) + 0×0) / (√2 × √2) = 2/2 = 1.0
```

유저A는 전반적으로 높은 점수, 유저B는 낮은 점수를 주지만 **선호 경향은 동일** → 유사도 1.0

#### 코사인 유사도와의 차이

| 구분 | 코사인 유사도 | 피어슨 상관계수 |
|------|-------------|----------------|
| 평균 보정 | 안 함 | 함 |
| 측정 대상 | 벡터의 방향 | 선호 경향의 상관관계 |
| 평점 스케일 차이 | 일부 반영 | 완전 제거 |
| 적합한 경우 | 암시적 피드백 | 명시적 피드백 (평점) |

---

### 3. 자카드 유사도 (Jaccard Similarity)

두 집합이 **얼마나 겹치는지**를 측정한다. 평점 값은 무시하고 **상호작용 여부만** 본다.

#### 수식

```
              |A ∩ B|
sim(A,B) = -----------
              |A ∪ B|
```

- 결과 범위: 0 ~ 1
- 1이면 완전 동일, 0이면 겹치는 것 없음

#### 예시

```
유저A가 본 영화 = {영화1, 영화2, 영화3, 영화5}
유저B가 본 영화 = {영화1, 영화3, 영화4, 영화5}

교집합 = {영화1, 영화3, 영화5} → 3개
합집합 = {영화1, 영화2, 영화3, 영화4, 영화5} → 5개

sim(A,B) = 3/5 = 0.6
```

#### 특징

- 이진 데이터(봤다/안봤다)에 적합
- 평점 값을 활용하지 못함
- 구매 이력, 클릭 이력 등 **암시적 피드백**에 자주 사용

---

### 4. 유클리드 거리 (Euclidean Distance)

두 벡터 사이의 **직선 거리**를 측정한다. 거리가 가까울수록 유사하다.

#### 수식

```
dist(A,B) = √Σ(Ai - Bi)²

유사도로 변환:
sim(A,B) = 1 / (1 + dist(A,B))
```

- 거리이므로 값이 작을수록 유사
- 유사도로 변환하면 0 ~ 1 범위

#### 예시

```
유저A = [5, 3, 4]
유저B = [4, 1, 2]

dist = √((5-4)² + (3-1)² + (4-2)²) = √(1 + 4 + 4) = √9 = 3

sim = 1 / (1 + 3) = 0.25
```

#### 특징

- 직관적이지만 **스케일에 민감** (정규화 필요)
- 고차원에서 거리가 뭉개지는 문제 (차원의 저주)
- 추천에서는 코사인/피어슨보다 덜 사용됨

---

## 유사도 선택 가이드

| 상황 | 추천 유사도 | 이유 |
|------|-----------|------|
| 평점 데이터 | 피어슨 상관계수 | 평점 스케일 차이를 보정 |
| 암시적 피드백 (횟수) | 코사인 유사도 | 크기보다 방향(패턴)이 중요 |
| 이진 데이터 (봤다/안봤다) | 자카드 유사도 | 집합 겹침이 핵심 |
| 저차원 연속 피처 | 유클리드 거리 | 실제 거리가 의미 있는 경우 |

실무에서는 **코사인 유사도**가 가장 범용적으로 쓰인다.

## 정리

```
유사도 측정 = "비슷하다"를 수학적으로 정의하는 것

- 코사인 유사도: 벡터 방향 비교, 가장 범용적
- 피어슨 상관계수: 평균 보정 후 경향 비교, 명시적 피드백에 적합
- 자카드 유사도: 집합 겹침 비교, 이진 데이터에 적합
- 유클리드 거리: 직선 거리, 스케일에 민감

→ 이 유사도들이 협업 필터링, 콘텐츠 기반 필터링의 기반이 된다
```
