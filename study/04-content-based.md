# 4. 콘텐츠 기반 필터링 (Content-Based Filtering)

## 핵심 아이디어

```
"사용자가 좋아한 아이템과 속성이 비슷한 아이템을 추천한다"
```

다른 사용자의 데이터가 필요 없다. 오직 **아이템의 속성**과 **해당 사용자의 이력**만 사용한다.

```
유저A가 액션 영화를 많이 봤다
→ 액션 속성을 가진 다른 영화를 추천
```

## 전체 흐름

```
1. 아이템 프로필 생성 — 각 아이템을 속성 벡터로 표현
2. 사용자 프로필 생성 — 사용자가 좋아한 아이템들의 속성을 종합
3. 유사도 계산       — 사용자 프로필과 후보 아이템 프로필의 유사도
4. 추천             — 유사도가 높은 아이템을 추천
```

---

## 1단계: 아이템 프로필 생성

아이템의 속성을 **벡터로 변환**하는 것이 핵심이다.

### 구조화된 속성의 경우

장르, 감독, 출시 연도 등 **이미 정해진 속성**이 있을 때

```
          액션  로맨스  코미디  SF   공포
영화A      1     0      1     0    0
영화B      0     1      0     0    0
영화C      1     0      0     1    0
영화D      0     0      1     1    0
```

### 텍스트 속성의 경우: TF-IDF

상품 설명, 기사 본문, 가사 등 **텍스트 데이터**를 벡터로 변환할 때 사용한다.

#### TF (Term Frequency) — 단어 빈도

해당 문서에서 특정 단어가 **얼마나 자주 등장하는가**

```
TF(t, d) = 문서 d에서 단어 t의 등장 횟수 / 문서 d의 전체 단어 수
```

#### IDF (Inverse Document Frequency) — 역문서 빈도

전체 문서에서 특정 단어가 **얼마나 희귀한가**

```
IDF(t) = log(전체 문서 수 / 단어 t가 등장한 문서 수)
```

- 모든 문서에 등장하는 단어 (예: "the", "이") → IDF 낮음
- 특정 문서에만 등장하는 단어 (예: "우주선", "좀비") → IDF 높음

#### TF-IDF

```
TF-IDF(t, d) = TF(t, d) × IDF(t)
```

"이 문서에서 자주 나오면서, 다른 문서에서는 잘 안 나오는 단어"에 높은 가중치를 부여한다.

#### 예시

```
문서1: "액션 영화 추천 액션 스릴러"
문서2: "로맨스 영화 감동 로맨스"
문서3: "액션 로맨스 영화"

단어 "액션":
  TF(문서1) = 2/5 = 0.4
  IDF = log(3/2) = 0.41    (3개 문서 중 2개에 등장)
  TF-IDF(문서1) = 0.4 × 0.41 = 0.16

단어 "스릴러":
  TF(문서1) = 1/5 = 0.2
  IDF = log(3/1) = 1.10    (3개 문서 중 1개에만 등장)
  TF-IDF(문서1) = 0.2 × 1.10 = 0.22

→ "스릴러"가 문서1을 더 잘 대표하는 단어
```

## 2단계: 사용자 프로필 생성

사용자가 좋아한 아이템들의 속성을 **평균 또는 가중 평균**하여 사용자 프로필을 만든다.

```
유저A가 좋아한 영화:
  영화A = [1, 0, 1, 0, 0]  (평점 5)
  영화C = [1, 0, 0, 1, 0]  (평점 4)
  영화D = [0, 0, 1, 1, 0]  (평점 3)

단순 평균:
  유저A 프로필 = [0.67, 0, 0.67, 0.67, 0]

가중 평균 (평점 반영):
  유저A 프로필 = (5×[1,0,1,0,0] + 4×[1,0,0,1,0] + 3×[0,0,1,1,0]) / (5+4+3)
               = [9, 0, 8, 7, 0] / 12
               = [0.75, 0, 0.67, 0.58, 0]
```

→ 유저A는 **액션 > 코미디 > SF** 순으로 선호

---

## 3단계: 유사도 계산 및 추천

사용자 프로필과 아직 보지 않은 아이템의 프로필 사이의 **코사인 유사도**를 계산한다.

```
유저A 프로필 = [0.75, 0, 0.67, 0.58, 0]

후보 아이템:
  영화E = [1, 0, 0, 1, 0]   → sim = ?  (액션 + SF)
  영화F = [0, 1, 0, 0, 1]   → sim = ?  (로맨스 + 공포)

코사인 유사도:
  sim(유저A, 영화E) = 높음 → 추천 ✓
  sim(유저A, 영화F) = 낮음 → 추천 ✗
```

## 장점과 단점

### 장점

- **Cold Start (아이템)에 강함** — 새 아이템도 속성만 있으면 바로 추천 가능
- **설명 가능** — "액션 장르를 좋아하셔서 추천합니다" 같은 설명이 가능
- **다른 사용자 데이터 불필요** — 사용자 한 명의 이력만으로 동작
- **투명성** — 왜 추천했는지 추적 가능

### 단점

- **다양성 부족 (필터 버블)** — 비슷한 것만 계속 추천, 새로운 발견 어려움
- **Cold Start (사용자)** — 이력이 없는 신규 사용자에게 추천 불가
- **속성 의존적** — 아이템 속성을 잘 정의해야 함, 속성 품질에 성능이 좌우됨
- **과적합** — 사용자의 과거 취향에만 갇힘

```
한계 예시:

유저가 액션 영화만 봤다면
→ 콘텐츠 기반은 계속 액션만 추천
→ 실제로는 안 봐서 모르는 것일 뿐, 로맨스도 좋아할 수 있음
→ 협업 필터링이 이 문제를 해결한다
```

## 정리

```
콘텐츠 기반 필터링 = 아이템 속성을 벡터화하여 사용자 취향과 비교

핵심 구성요소:
- 아이템 프로필: 속성을 벡터로 변환 (구조화 데이터 or TF-IDF)
- 사용자 프로필: 좋아한 아이템 속성의 가중 평균
- 유사도: 코사인 유사도로 매칭

강점: 설명 가능, 새 아이템 추천 가능
약점: 다양성 부족, 필터 버블

→ 다음 장에서 다루는 협업 필터링이 이 한계를 보완한다
```
