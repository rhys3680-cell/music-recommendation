# 5. 협업 필터링 - 메모리 기반 (Memory-Based Collaborative Filtering)

## 핵심 아이디어

```
"아이템의 속성은 몰라도 된다. 사용자들의 행동 패턴만으로 추천한다."
```

콘텐츠 기반 필터링은 "이 영화는 액션이다"라는 속성이 필요했지만, 협업 필터링은 **누가 뭘 좋아했는지만** 있으면 된다.

```
유저A와 유저B가 영화1, 영화2, 영화3을 모두 좋아했다
유저B는 영화4도 좋아했다
→ 유저A에게 영화4를 추천
```

---

## 메모리 기반 vs 모델 기반

협업 필터링은 크게 두 갈래로 나뉜다.

```
협업 필터링
├── 메모리 기반 (이번 장) — 원본 데이터를 직접 사용, 유사도 계산
│   ├── User-Based CF
│   └── Item-Based CF
└── 모델 기반 (다음 장) — 데이터에서 모델을 학습 (MF, SVD 등)
```

메모리 기반은 **사용자-아이템 행렬 원본을 그대로 보관(메모리)하고**, 추천 시점에 유사도를 계산한다.

---

## User-Based CF

### 원리

```
1. 나와 비슷한 사용자(이웃)를 찾는다
2. 이웃이 좋아했지만 나는 아직 안 본 아이템을 추천한다
```

### 단계별 과정

#### 1단계: 사용자-아이템 행렬

```
        영화1  영화2  영화3  영화4  영화5
유저A     5      3      4      ?      ?
유저B     4      ?      5      4      ?
유저C     ?      3      ?      4      5
유저D     5      4      4      ?      3

목표: 유저A의 영화4 예측 평점 = ?
```

#### 2단계: 유저A와 다른 사용자의 유사도 계산

공통으로 평가한 아이템만 사용하여 유사도를 계산한다.

```
유저A vs 유저B (공통: 영화1, 영화3)
  A = [5, 4]
  B = [4, 5]
  코사인 유사도 = (5×4 + 4×5) / (√41 × √41) = 40/41 ≈ 0.98

유저A vs 유저C (공통: 영화2)
  A = [3]
  B = [3]
  코사인 유사도 = 1.0  (데이터가 너무 적어 신뢰도 낮음)

유저A vs 유저D (공통: 영화1, 영화2, 영화3)
  A = [5, 3, 4]
  D = [5, 4, 4]
  코사인 유사도 ≈ 0.99
```

#### 3단계: 이웃 선택

유사도가 높은 상위 K명을 이웃으로 선택한다.

```
K = 2로 설정
이웃: 유저D (0.99), 유저B (0.98)
```

#### 4단계: 평점 예측

이웃의 평점을 **유사도로 가중 평균**하여 예측한다.

```
기본 공식:
                Σ(sim(A, u) × rating(u, i))
pred(A, i) = ──────────────────────────────
                    Σ|sim(A, u)|
```

```
유저A의 영화4 예측:

유저B: sim = 0.98, 영화4 평점 = 4
유저D: sim = 0.99, 영화4 평점 = ?  → 평가 안 함, 제외

→ 유저B만 영화4를 평가함
pred(A, 영화4) = (0.98 × 4) / 0.98 = 4.0
```

#### 평균 보정 버전

사용자마다 평점 기준이 다르므로 **평균을 빼고** 계산하면 더 정확하다.

```
                          Σ(sim(A, u) × (rating(u, i) - r̄u))
pred(A, i) = r̄A + ──────────────────────────────────────
                              Σ|sim(A, u)|

r̄A = 유저A의 평균 평점
r̄u = 이웃 u의 평균 평점
```

```
유저A 평균 = (5+3+4)/3 = 4.0
유저B 평균 = (4+5+4)/3 = 4.33

pred(A, 영화4) = 4.0 + (0.98 × (4 - 4.33)) / 0.98
               = 4.0 + (-0.33)
               = 3.67
```

→ 유저B는 평균보다 낮게 영화4를 평가했으므로, 보정 후 예측이 낮아진다.

## Item-Based CF

### 원리

```
1. 내가 좋아한 아이템과 비슷한 아이템을 찾는다
2. 비슷한 아이템을 추천한다
```

User-Based는 "비슷한 사용자"를 찾지만, Item-Based는 **"비슷한 아이템"**을 찾는다.

### 아이템 유사도 계산

아이템을 벡터로 표현할 때, **해당 아이템을 평가한 사용자들의 평점 벡터**를 사용한다.

```
        유저A  유저B  유저C  유저D
영화1     5      4      ?      5       ← 영화1의 벡터
영화2     3      ?      3      4       ← 영화2의 벡터
영화3     4      5      ?      4
영화4     ?      4      4      ?
영화5     ?      ?      5      3

영화1 vs 영화4 유사도 (공통 평가 유저: 유저B)
  영화1 = [4]
  영화4 = [4]
  sim = 1.0

영화3 vs 영화4 유사도 (공통 평가 유저: 유저B)
  영화3 = [5]
  영화4 = [4]
  sim = ... (데이터 1개로는 부정확)
```

### 평점 예측

```
                Σ(sim(i, j) × rating(A, j))
pred(A, i) = ──────────────────────────────
                    Σ|sim(i, j)|

j = 유저A가 이미 평가한 아이템 중 아이템 i와 유사한 것들
```

## User-Based vs Item-Based 비교

| 구분 | User-Based | Item-Based |
|------|-----------|-----------|
| 유사도 대상 | 사용자 간 유사도 | 아이템 간 유사도 |
| 관점 | "나와 비슷한 사람이 좋아한 것" | "내가 좋아한 것과 비슷한 것" |
| 아이템 수 << 사용자 수 | 느림 | **빠름** |
| 사용자 수 << 아이템 수 | **빠름** | 느림 |
| 안정성 | 낮음 (사용자 취향 변함) | **높음** (아이템 속성 고정) |
| 실무 선호 | 덜 사용됨 | **더 많이 사용됨** |
| 설명 가능성 | "비슷한 취향의 사용자도 좋아했습니다" | "이 상품과 비슷한 상품입니다" |

### 왜 Item-Based가 더 많이 쓰이는가?

```
1. 안정성: 아이템 관계는 잘 안 변하지만, 사용자 취향은 자주 변한다
2. 확장성: 보통 아이템 수 < 사용자 수 → 아이템 유사도 행렬이 더 작다
3. 사전 계산: 아이템 유사도를 미리 계산해두면 추천 시점에 빠르다
4. 설명: "이 상품을 본 사람은 이 상품도 봤습니다" — Amazon의 대표 기능
```

---

## 이웃 선택 방법

### K-NN (K Nearest Neighbors)

유사도 상위 K개만 이웃으로 선택한다.

```
K가 너무 작으면 → 노이즈에 민감, 과적합
K가 너무 크면   → 관련 없는 이웃 포함, 성능 저하

일반적으로 K = 20~50 사이에서 조정
```

### 임계값 기반 (Threshold)

유사도가 특정 값 이상인 것만 이웃으로 선택한다.

```
sim > 0.5 인 사용자/아이템만 이웃으로 사용
→ 이웃 수가 유동적
```

## 메모리 기반 CF의 한계

### 1. 데이터 희소성 (Sparsity)

```
실제 서비스 데이터:
- Netflix: 48만 사용자 × 1.7만 영화 → 행렬의 약 1%만 채워져 있음
- 대부분의 칸이 비어있어 유사도 계산이 부정확
```

### 2. 확장성 (Scalability)

```
사용자 100만 명 × 아이템 10만 개
→ User-Based: 100만 × 100만 유사도 계산 필요
→ 실시간 계산 불가능
```

### 3. Cold Start

```
새 사용자: 평가 이력이 없어 유사도 계산 불가
새 아이템: 아무도 평가하지 않아 추천 대상에서 제외
```

### 4. 인기 편향

```
인기 아이템은 많은 사용자와 겹치므로 유사도가 높게 나옴
→ 인기 아이템만 반복 추천되는 문제
```

이 한계들을 해결하기 위해 **모델 기반 협업 필터링** (Matrix Factorization 등)이 등장한다.

---

## 정리

```
메모리 기반 CF = 원본 데이터에서 직접 유사도를 계산하여 추천

User-Based CF:
  "나와 비슷한 사용자가 좋아한 것을 추천"
  → 유사한 사용자 찾기 → 그들의 평점으로 예측

Item-Based CF:
  "내가 좋아한 아이템과 비슷한 아이템을 추천"
  → 유사한 아이템 찾기 → 내 평점으로 예측
  → 실무에서 더 선호 (안정적, 설명 가능)

한계: 희소성, 확장성, Cold Start
→ 다음 장의 모델 기반 CF (Matrix Factorization)가 이를 보완
```
