# 6. 협업 필터링 - 모델 기반 (Model-Based Collaborative Filtering)

## 왜 모델 기반이 필요한가?

메모리 기반 CF의 한계를 다시 보면:

```
1. 희소성 — 행렬의 99%가 비어있어 유사도가 부정확
2. 확장성 — 사용자 100만 명이면 유사도 계산이 불가능
3. 일반화 — 원본 데이터를 그대로 쓰니 패턴을 일반화하지 못함
```

모델 기반은 원본 행렬에서 **숨겨진 패턴(잠재 요인)을 학습**하여 이 문제들을 해결한다.

---

## Matrix Factorization (행렬 분해)

### 핵심 아이디어

```
거대한 사용자-아이템 행렬을
두 개의 작은 행렬의 곱으로 분해한다

R ≈ P × Q^T

R: 사용자-아이템 행렬 (m × n)
P: 사용자 잠재 요인 행렬 (m × k)
Q: 아이템 잠재 요인 행렬 (n × k)
k: 잠재 요인 수 (보통 20~200)
```

### 잠재 요인 (Latent Factor)이란?

명시적으로 정의하지 않아도 데이터에서 **자동으로 발견되는 숨겨진 특성**이다.

```
영화 예시에서 잠재 요인이 자동으로 발견할 수 있는 것들:

요인1: 액션 ←→ 로맨스 스펙트럼
요인2: 가벼운 ←→ 무거운 스펙트럼
요인3: 오래된 ←→ 최신 스펙트럼
...

이 요인들은 사람이 정의하는 게 아니라 데이터에서 학습된다
→ 해석이 직관적이지 않을 수도 있음
```

### 시각적으로 이해하기

```
원본 행렬 R (100만 유저 × 10만 아이템)

        아이템1  아이템2  아이템3  ...  아이템10만
유저1     5       ?       3      ...    ?
유저2     ?       4       ?      ...    2
유저3     3       ?       ?      ...    ?
...
유저100만  ?       ?       5      ...    ?

→ 99%가 비어있는 거대한 행렬


분해 후:

P (100만 × k)          Q^T (k × 10만)
유저 잠재 벡터          아이템 잠재 벡터

유저1  [0.2, 1.5, 0.8]     [0.5, 0.3, 1.2]  아이템1
유저2  [1.1, 0.3, 0.6]  ×  [1.0, 0.1, 0.7]  아이템2
유저3  [0.8, 1.2, 0.4]     [0.3, 1.4, 0.2]  아이템3
...                         ...

예측: R̂(유저1, 아이템2) = P[1] · Q[2]
    = 0.2×1.0 + 1.5×0.1 + 0.8×0.7
    = 0.2 + 0.15 + 0.56 = 0.91
```

k=3이면 각 사용자와 아이템을 **3차원 벡터**로 표현한 것이다. 원래 10만 차원이 필요했던 것을 3차원으로 압축한 셈이다.

## SVD (Singular Value Decomposition)

### 기본 SVD

선형대수의 행렬 분해 방법이다.

```
R = U × Σ × V^T

U: 좌측 특이 벡터 (사용자 방향)
Σ: 특이값 대각 행렬 (중요도)
V: 우측 특이 벡터 (아이템 방향)
```

#### 문제점

```
기본 SVD는 행렬에 빈 칸이 없어야 한다.
하지만 추천 행렬은 99%가 빈 칸이다.
→ 빈 칸을 0이나 평균으로 채우면 노이즈가 심해짐
→ 그래서 FunkSVD가 등장
```

### FunkSVD (Simon Funk의 SVD)

Netflix Prize에서 사용되어 유명해진 방법이다. 빈 칸 문제를 해결했다.

```
핵심: 빈 칸을 채우지 않고, 관측된 값만으로 학습한다

예측:
R̂(u, i) = P_u · Q_i = Σ(p_uk × q_ki)   (k = 1 ~ K)

학습 목표:
관측된 (u, i) 쌍에 대해서만 오차를 최소화

min Σ (r_ui - P_u · Q_i)² + λ(|P_u|² + |Q_i|²)
     (u,i)∈관측값

λ(|P_u|² + |Q_i|²) = 정규화 항 (과적합 방지)
```

### 경사 하강법 (SGD)으로 학습

```
각 관측된 평점 (u, i, r_ui)에 대해:

1. 예측: R̂ = P_u · Q_i
2. 오차: e = r_ui - R̂
3. 업데이트:
   P_u ← P_u + α(e × Q_i - λ × P_u)
   Q_i ← Q_i + α(e × P_u - λ × Q_i)

α = 학습률 (보통 0.005~0.01)
λ = 정규화 계수 (보통 0.02~0.1)
```

#### 학습 과정 예시

```
초기 (랜덤):
P_유저1 = [0.1, 0.2]
Q_영화3 = [0.3, 0.4]

실제 평점: r = 5
예측: R̂ = 0.1×0.3 + 0.2×0.4 = 0.11
오차: e = 5 - 0.11 = 4.89  (매우 큼)

업데이트 (α=0.01, λ=0.02):
P_유저1 ← [0.1, 0.2] + 0.01 × (4.89×[0.3, 0.4] - 0.02×[0.1, 0.2])
        = [0.1, 0.2] + [0.0147, 0.0196]
        ≈ [0.115, 0.220]

→ 반복하면서 P와 Q가 점점 실제 평점에 가까워진다
```

### 편향 추가 (Biased MF)

사용자마다, 아이템마다 기본적인 편향이 있다.

```
예측 = 전체 평균 + 사용자 편향 + 아이템 편향 + 상호작용

R̂(u, i) = μ + b_u + b_i + P_u · Q_i

μ   = 전체 평균 평점 (예: 3.5)
b_u = 사용자 편향 (후한 사용자 +0.5, 짠 사용자 -0.3)
b_i = 아이템 편향 (인기작 +1.0, 비인기작 -0.5)
P_u · Q_i = 개인 취향과 아이템 특성의 상호작용
```

```
예시:
전체 평균 μ = 3.5
유저A 편향 b_u = +0.3 (평균보다 후하게 줌)
영화X 편향 b_i = +0.8 (전반적으로 인기 많은 영화)
P_유저A · Q_영화X = +0.4 (유저A의 취향과 잘 맞음)

R̂ = 3.5 + 0.3 + 0.8 + 0.4 = 5.0
```

## ALS (Alternating Least Squares)

### SGD vs ALS

```
SGD: 하나의 (유저, 아이템) 쌍씩 순차적으로 업데이트
     → 구현 간단, 수렴 빠름
     → 병렬화 어려움

ALS: P를 고정하고 Q를 최적화 → Q를 고정하고 P를 최적화 → 반복
     → 각 단계가 최소제곱 문제 → 정확한 해를 구할 수 있음
     → 병렬화 가능 → 대규모 데이터에 유리
```

### ALS 동작 원리

```
1. P, Q를 랜덤으로 초기화

2. P를 고정하고 Q를 최적화:
   각 아이템 i에 대해:
   Q_i = (P^T × P + λI)^(-1) × P^T × R_i

3. Q를 고정하고 P를 최적화:
   각 유저 u에 대해:
   P_u = (Q^T × Q + λI)^(-1) × Q^T × R_u

4. 수렴할 때까지 2-3 반복
```

### 언제 어떤 것을 쓰나?

| 구분 | SGD | ALS |
|------|-----|-----|
| 명시적 피드백 | **적합** | 적합 |
| 암시적 피드백 | 가능 | **더 적합** |
| 소규모 데이터 | **빠름** | 오버헤드 |
| 대규모 데이터 | 느림 | **병렬화 가능** |
| 구현 난이도 | 쉬움 | 보통 |
| Spark 등 분산환경 | 어려움 | **적합** |

## 메모리 기반 vs 모델 기반 비교

| 구분 | 메모리 기반 | 모델 기반 (MF) |
|------|-----------|---------------|
| 데이터 사용 | 원본 그대로 | 학습된 모델 |
| 희소성 처리 | 약함 | **강함** (잠재 요인으로 일반화) |
| 확장성 | 나쁨 | **좋음** (저차원 벡터) |
| 학습 시간 | 없음 (실시간 계산) | 있음 (오프라인 학습) |
| 추천 속도 | 느림 | **빠름** (벡터 내적) |
| 설명 가능성 | 높음 | 낮음 (잠재 요인 해석 어려움) |
| 새 데이터 반영 | 즉시 | 재학습 필요 |

## 하이퍼파라미터 튜닝

```
k (잠재 요인 수):
  너무 작으면 → 표현력 부족, 과소적합
  너무 크면   → 과적합, 계산량 증가
  일반적: 20 ~ 200

α (학습률, SGD):
  너무 크면 → 발산
  너무 작으면 → 수렴 느림
  일반적: 0.001 ~ 0.01

λ (정규화):
  너무 크면 → 과소적합
  너무 작으면 → 과적합
  일반적: 0.01 ~ 0.1

반복 횟수:
  일반적: 20 ~ 100 에폭
```

## 정리

```
모델 기반 CF = 사용자-아이템 행렬을 저차원 잠재 요인으로 분해

Matrix Factorization:
  R ≈ P × Q^T
  → 사용자와 아이템을 k차원 벡터로 표현
  → 희소성 문제 해결, 일반화 능력 확보

학습 방법:
  SGD: 순차 업데이트, 구현 간단, 소규모에 적합
  ALS: 번갈아 최적화, 병렬화 가능, 대규모에 적합

핵심 개선:
  편향 추가 (μ + b_u + b_i + P·Q)로 예측 정확도 향상

한계:
  잠재 요인 해석 어려움, 재학습 필요, 선형 관계만 포착
  → 딥러닝 기반 추천이 이를 보완 (Phase 3에서 다룸)
```
