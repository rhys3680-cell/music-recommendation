# 3. 평가 지표

추천 시스템을 만들었으면 **"얼마나 잘 추천하는가?"**를 측정해야 한다. 목적에 따라 다른 지표를 사용한다.

## 평가 지표의 분류

```
1. 정확도 지표 — 예측 값이 실제 값과 얼마나 가까운가?
2. 랭킹 지표  — 상위에 노출된 아이템이 실제로 좋은가?
3. 비정확도 지표 — 다양성, 참신성, 커버리지 등
```

---

## 1. 정확도 지표

예측 평점과 실제 평점의 차이를 측정한다. 주로 **평점 예측 문제**에 사용한다.

### MAE (Mean Absolute Error)

예측과 실제의 **절대 오차 평균**

```
MAE = (1/n) × Σ|실제i - 예측i|
```

#### 예시

```
실제 평점:  [5, 3, 4, 2]
예측 평점:  [4, 3, 5, 1]

오차:      [1, 0, 1, 1]

MAE = (1 + 0 + 1 + 1) / 4 = 0.75
```

- 직관적이고 해석이 쉬움
- 모든 오차를 동일하게 취급

### RMSE (Root Mean Squared Error)

예측과 실제의 **제곱 오차 평균의 제곱근**

```
RMSE = √((1/n) × Σ(실제i - 예측i)²)
```

#### 예시

```
실제 평점:  [5, 3, 4, 2]
예측 평점:  [4, 3, 5, 1]

제곱 오차:  [1, 0, 1, 1]

RMSE = √((1 + 0 + 1 + 1) / 4) = √0.75 ≈ 0.87
```

- **큰 오차에 더 큰 페널티**를 부여 (제곱 때문에)
- Netflix Prize에서 사용된 대표 지표

### MAE vs RMSE

```
실제: [5, 3]
예측: [4, 1]   → 오차: [1, 2]

MAE  = (1 + 2) / 2 = 1.5
RMSE = √((1 + 4) / 2) = √2.5 ≈ 1.58

→ RMSE가 더 크다 = 큰 오차(2점)에 더 민감하게 반응
```

## 2. 분류 지표

추천을 **"맞았다/틀렸다"**로 판단한다. 주로 Top-K 추천 목록 평가에 사용한다.

### 혼동 행렬 (Confusion Matrix)

```
                    실제 관심 있음    실제 관심 없음
추천함 (Positive)      TP               FP
추천 안 함 (Negative)   FN               TN

TP = True Positive   (추천했고, 실제로 좋아함) ✓
FP = False Positive  (추천했지만, 관심 없음)   — 잘못된 추천
FN = False Negative  (추천 안 했지만, 좋아했을 것) — 놓친 추천
TN = True Negative   (추천 안 했고, 실제로 관심 없음) ✓
```

### Precision (정밀도)

추천한 것 중 **실제로 관심 있는 비율**

```
Precision = TP / (TP + FP)

"추천한 10개 중 몇 개를 사용자가 실제로 좋아했나?"
```

#### 예시

```
추천 목록 (5개): [A, B, C, D, E]
실제 관심:       [A, C, F, G]

맞은 것: A, C → TP = 2
틀린 것: B, D, E → FP = 3

Precision@5 = 2 / 5 = 0.4
```

### Recall (재현율)

관심 있는 것 중 **추천 목록에 포함된 비율**

```
Recall = TP / (TP + FN)

"사용자가 좋아할 아이템 중 몇 개를 추천 목록에 넣었나?"
```

#### 예시 (위와 같은 데이터)

```
추천 목록 (5개): [A, B, C, D, E]
실제 관심:       [A, C, F, G]

맞은 것: A, C → TP = 2
놓친 것: F, G → FN = 2

Recall@5 = 2 / 4 = 0.5
```

### Precision vs Recall 트레이드오프

```
추천 많이 하면 → Recall ↑ Precision ↓ (많이 추천하니 놓치는 건 줄지만 틀리는 것도 많아짐)
추천 적게 하면 → Precision ↑ Recall ↓ (확실한 것만 추천하니 정확하지만 놓치는 게 많아짐)
```

### F1 Score

Precision과 Recall의 **조화 평균** — 둘 다 균형 있게 높아야 점수가 높다.

```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

```
Precision = 0.4, Recall = 0.5

F1 = 2 × (0.4 × 0.5) / (0.4 + 0.5) = 0.4 / 0.9 ≈ 0.44
```

## 3. 랭킹 지표

추천은 단순히 "맞았다/틀렸다"보다 **"좋은 아이템이 상위에 있는가?"**가 더 중요하다.

### Hit Rate (HR@K)

Top-K 추천 목록에 **관심 아이템이 하나라도 있으면 Hit**

```
HR@K = Hit한 사용자 수 / 전체 사용자 수
```

```
유저1: 추천 [A,B,C,D,E] → 실제 관심 A → Hit ✓
유저2: 추천 [A,B,C,D,E] → 실제 관심 F → Miss ✗
유저3: 추천 [A,B,C,D,E] → 실제 관심 C → Hit ✓

HR@5 = 2/3 ≈ 0.67
```

### MRR (Mean Reciprocal Rank)

첫 번째 관련 아이템의 **순위의 역수** 평균

```
MRR = (1/|U|) × Σ(1 / ranki)
```

```
유저1: 추천 [A,B,C] → 관심 아이템 A가 1번째 → 1/1 = 1.0
유저2: 추천 [A,B,C] → 관심 아이템 C가 3번째 → 1/3 = 0.33
유저3: 추천 [A,B,C] → 관심 아이템 없음        → 0

MRR = (1.0 + 0.33 + 0) / 3 ≈ 0.44
```

- "좋은 추천이 **빨리** 나오는가?"를 측정
- 검색 엔진에서도 많이 사용

### MAP (Mean Average Precision)

여러 관련 아이템이 있을 때, 각 관련 아이템 위치에서의 **Precision 평균**

```
AP = (1/관련 아이템 수) × Σ(Precision@k × rel(k))
MAP = 전체 사용자의 AP 평균
```

```
추천 목록: [A, B, C, D, E]
관련 아이템: {A, C, E}

위치 1 (A, 관련 ✓): Precision@1 = 1/1 = 1.0
위치 2 (B, 무관 ✗): 스킵
위치 3 (C, 관련 ✓): Precision@3 = 2/3 = 0.67
위치 4 (D, 무관 ✗): 스킵
위치 5 (E, 관련 ✓): Precision@5 = 3/5 = 0.6

AP = (1.0 + 0.67 + 0.6) / 3 = 0.76
```

### NDCG (Normalized Discounted Cumulative Gain)

아이템마다 **관련도 점수가 다를 때** 순위를 평가한다. 상위에 높은 관련도 아이템이 올수록 점수가 높다.

```
DCG@K = Σ(reli / log₂(i + 1))     (i = 1부터 K까지)
IDCG@K = 이상적 순서로 정렬했을 때의 DCG
NDCG@K = DCG@K / IDCG@K
```

#### 예시

```
추천 순서:    [아이템A, 아이템B, 아이템C, 아이템D]
관련도 점수:  [3,       1,       2,       0     ]

DCG@4 = 3/log₂(2) + 1/log₂(3) + 2/log₂(4) + 0/log₂(5)
      = 3/1 + 1/1.58 + 2/2 + 0
      = 3 + 0.63 + 1 + 0
      = 4.63

이상적 순서:  [3, 2, 1, 0]   (내림차순)
IDCG@4 = 3/1 + 2/1.58 + 1/2 + 0
       = 3 + 1.26 + 0.5 + 0
       = 4.76

NDCG@4 = 4.63 / 4.76 ≈ 0.97
```

- 1에 가까울수록 이상적인 순서에 가까움
- 추천 시스템에서 **가장 중요한 랭킹 지표**

## 4. 비정확도 지표

정확도만으로는 좋은 추천 시스템이 아니다. 다양성과 참신성도 중요하다.

### Coverage (커버리지)

전체 아이템 중 **추천 목록에 한 번이라도 등장한 아이템의 비율**

```
Coverage = 추천된 고유 아이템 수 / 전체 아이템 수
```

- 낮으면 인기 아이템만 반복 추천하는 것

### Novelty (참신성)

추천된 아이템이 **얼마나 덜 알려진 것인가**

```
인기 아이템 추천 → 참신성 낮음 (누구나 아는 것)
비인기 아이템 추천 → 참신성 높음 (새로운 발견)
```

### Serendipity (의외성)

사용자가 **예상하지 못했지만 좋아하는 추천**

```
항상 듣던 장르만 추천 → 의외성 낮음
새로운 장르인데 마음에 듦 → 의외성 높음
```

---

## 지표 선택 가이드

| 상황 | 추천 지표 | 이유 |
|------|----------|------|
| 평점 예측 | RMSE, MAE | 예측 정확도 직접 측정 |
| Top-K 추천 목록 | Precision@K, Recall@K | 추천 목록의 품질 |
| 순위가 중요할 때 | NDCG, MAP | 상위 노출 품질 |
| 첫 결과가 중요할 때 | MRR, HR | 검색/1개 추천 시나리오 |
| 서비스 품질 전반 | Coverage + Novelty | 다양성 확보 |

실무에서는 **NDCG + Recall@K** 조합이 가장 많이 쓰인다.

## 정리

```
평가 지표 = 추천 시스템이 얼마나 잘 작동하는지 측정하는 도구

- 정확도: MAE, RMSE → 평점 예측 오차
- 분류: Precision, Recall, F1 → 맞았다/틀렸다
- 랭킹: NDCG, MAP, MRR → 좋은 것이 위에 있는가?
- 다양성: Coverage, Novelty, Serendipity → 뻔하지 않은가?

→ 하나의 지표만으로는 부족하다. 여러 지표를 조합해서 평가해야 한다.
```
