# 1. 추천 시스템 개요

## 추천 시스템이란?

사용자에게 **수많은 아이템 중에서 관심 있을 만한 것을 골라주는 시스템**이다.

핵심 문제를 한 줄로 표현하면:

```
"사용자 U가 아이템 I에 대해 얼마나 좋아할지 예측하라"
→ f(U, I) = 예측 선호도
```

## 어디에 쓰이나?

| 서비스 | 추천 대상 | 시그널 |
|--------|----------|--------|
| Netflix | 영화, 드라마 | 시청 기록, 평점 |
| Spotify | 노래, 플레이리스트 | 재생, 스킵, 저장 |
| Amazon | 상품 | 구매, 클릭, 장바구니 |
| YouTube | 영상 | 시청 시간, 좋아요 |
| Instagram | 게시물, 릴스 | 좋아요, 저장, 체류 시간 |

공통점: **사용자 행동 데이터 → 선호도 예측 → 아이템 추천**

## 피드백의 두 가지 유형

### 명시적 피드백 (Explicit Feedback)

사용자가 **직접 표현한** 선호도

- 별점 (1~5점)
- 좋아요 / 싫어요
- 리뷰 작성

```
사용자-아이템 행렬 (명시적)

        영화A  영화B  영화C  영화D
유저1     5      3      ?      1
유저2     4      ?      ?      5
유저3     ?      1      5      ?
유저4     ?      ?      4      4

? = 아직 평가 안 함 → 이걸 예측하는 게 추천의 목표
```

- **장점**: 선호가 명확함
- **단점**: 데이터가 적음 — 대부분의 사용자는 평점을 안 남김

### 암시적 피드백 (Implicit Feedback)

사용자 **행동에서 추론한** 선호도

- 클릭, 조회
- 재생 시간, 스킵 여부
- 구매, 장바구니 담기
- 검색 기록

```
사용자-아이템 행렬 (암시적)

        곡A   곡B   곡C   곡D
유저1    32     1     0     5    ← 재생 횟수
유저2    0     15     0    28
유저3    10     0    45     0

0 = 안 들은 건지, 모르는 건지, 싫은 건지 구분 불가
```

- **장점**: 데이터가 풍부함
- **단점**: 노이즈가 많고, "안 본 것 ≠ 싫어하는 것" 구분이 어려움

### 비교

| 구분 | 명시적 | 암시적 |
|------|--------|--------|
| 데이터 양 | 적음 | 많음 |
| 신호 품질 | 높음 | 낮음 (노이즈) |
| 부정 신호 | 있음 (1점, 싫어요) | 없거나 약함 |
| 실제 서비스 | 보조적 활용 | **주로 사용** |

현대 추천 시스템은 **암시적 피드백 중심**으로 동작한다. 명시적 피드백은 보조 시그널로 활용한다.

## 추천 시스템의 3가지 접근 방식

```
1. 콘텐츠 기반 필터링 (CBF)
   "이 아이템과 비슷한 속성의 아이템을 추천"

2. 협업 필터링 (CF)
   "나와 비슷한 사용자가 좋아한 아이템을 추천"

3. 하이브리드
   "둘 다 쓰자"
```

각 접근 방식은 Phase 2에서 하나씩 깊이 다룬다.

## 추천 시스템의 핵심 과제

- **Cold Start** — 새 사용자/아이템은 데이터가 없어서 추천 불가
- **확장성** — 아이템이 수백만 개일 때 어떻게 빠르게 계산할 것인가
- **다양성 vs 정확도** — 정확한 추천만 하면 비슷한 것만 보여줌 (필터 버블)
- **데이터 희소성** — 대부분의 사용자-아이템 조합은 비어 있음

## 정리

```
추천 시스템 = 사용자-아이템 상호작용 데이터로 선호도를 예측하는 시스템

- 피드백: 명시적 (평점) vs 암시적 (행동) → 실제로는 암시적이 주력
- 접근법: 콘텐츠 기반 / 협업 필터링 / 하이브리드
- 과제: Cold Start, 확장성, 다양성, 희소성
```
