# 9. 딥러닝 기반 추천 (Deep Learning for Recommendation)

## 왜 딥러닝인가?

기존 MF의 한계:

```
MF: R̂(u, i) = P_u · Q_i (내적)

→ 사용자와 아이템의 관계를 "선형 결합"으로만 표현
→ 복잡한 비선형 패턴을 포착하지 못함

예시:
  유저A는 "액션이면서 코미디인 영화"를 좋아하지만
  "액션만" 또는 "코미디만"인 영화는 별로
  → 이런 조합 효과를 내적으로는 표현하기 어려움
```

딥러닝은 **비선형 관계를 학습**할 수 있어 이 한계를 넘는다.

---

## Neural Collaborative Filtering (NCF)

He et al. (2017). MF의 내적을 **신경망으로 대체**한 모델이다.

### 구조

```
입력:  유저 ID → 유저 임베딩
       아이템 ID → 아이템 임베딩

           유저 임베딩    아이템 임베딩
               \            /
                \          /
               [  결합 (concat)  ]
                      |
               [ Hidden Layer 1 ] → ReLU
                      |
               [ Hidden Layer 2 ] → ReLU
                      |
               [ Hidden Layer 3 ] → ReLU
                      |
               [   출력 Layer   ] → Sigmoid
                      |
                  예측 점수 (0~1)
```

### MF vs NCF 비교

```
MF:
  score = P_u · Q_i
  → 내적 하나로 끝

NCF:
  score = f(P_u, Q_i)
  → 신경망이 유저-아이템 관계를 비선형으로 학습
  → 더 복잡한 패턴 포착 가능
```

### NeuMF (Neural Matrix Factorization)

MF와 MLP를 **동시에 사용**하여 결합하는 모델이다.

```
유저 임베딩 ──┬── MF 경로: 내적 계산 ──────────┐
              │                                 │
              └── MLP 경로: 다층 신경망 ─────────┤
                                                │
아이템 임베딩 ──┬── MF 경로: 내적 계산 ──────────┤
               │                                │
               └── MLP 경로: 다층 신경망 ────────┤
                                                │
                                         [ concat ]
                                              |
                                        [ 출력 Layer ]
                                              |
                                          예측 점수

→ MF의 선형 관계 + MLP의 비선형 관계를 모두 포착
```

## Embedding (임베딩)

딥러닝 추천의 핵심 구성요소이다. ID를 **밀집 벡터(dense vector)**로 변환한다.

### 왜 임베딩이 필요한가?

```
One-hot 인코딩:
  유저1 = [1, 0, 0, 0, ..., 0]   (100만 차원)
  유저2 = [0, 1, 0, 0, ..., 0]
  → 차원이 너무 크고, 유저 간 관계 정보 없음

임베딩:
  유저1 = [0.2, 1.5, -0.3, ..., 0.8]   (128차원)
  유저2 = [0.3, 1.4, -0.2, ..., 0.7]
  → 저차원, 비슷한 유저는 비슷한 벡터
  → 학습을 통해 자동으로 의미 있는 표현을 얻음
```

### 임베딩이 학습되는 과정

```
초기: 랜덤 벡터
  유저A = [0.1, -0.5, 0.3]
  곡X   = [0.8, 0.2, -0.1]

학습 중: 유저A가 곡X를 좋아함 → 두 벡터가 가까워지도록 업데이트
  유저A = [0.3, -0.2, 0.1]
  곡X   = [0.5, 0.0, 0.1]

학습 후: 비슷한 취향의 유저 → 비슷한 벡터
         비슷한 곡 → 비슷한 벡터
```

### 다양한 피처의 임베딩

ID뿐 아니라 **다양한 범주형 피처**를 임베딩할 수 있다.

```
유저 측:
  유저 ID     → 임베딩 (64차원)
  성별        → 임베딩 (4차원)
  연령대      → 임베딩 (8차원)
  국가        → 임베딩 (16차원)

아이템 측:
  아이템 ID   → 임베딩 (64차원)
  장르        → 임베딩 (16차원)
  아티스트    → 임베딩 (32차원)

→ 모든 임베딩을 concat하여 신경망 입력으로 사용
```

## Autoencoder 기반 추천

사용자의 상호작용 벡터를 **압축했다가 복원**하면서 빈 칸을 채운다.

### 구조

```
입력: 유저A의 평점 벡터 [5, ?, 3, ?, 4, ?]
      (빈 칸은 0으로)  [5, 0, 3, 0, 4, 0]

      [5, 0, 3, 0, 4, 0]   ← 입력 (n차원)
             |
      [ Encoder 층 ]
             |
      [  잠재 표현  ]        ← 저차원 (k차원)
             |
      [ Decoder 층 ]
             |
      [4.9, 3.2, 3.1, 2.8, 3.9, 4.1]  ← 복원 (n차원)

→ 복원된 값에서 원래 빈 칸이었던 곳의 값이 예측 평점
→ 유저A의 아이템2 예측 = 3.2, 아이템4 예측 = 2.8, 아이템6 예측 = 4.1
```

### 왜 작동하는가?

```
Encoder가 "이 유저의 취향 핵심"을 저차원으로 압축
→ MF의 잠재 요인과 비슷한 역할
→ 하지만 비선형 변환이 가능하므로 더 복잡한 패턴 포착

Decoder가 압축된 취향으로부터 "모든 아이템에 대한 선호"를 복원
→ 빈 칸이 자연스럽게 채워짐
```

### 변형: Variational Autoencoder (VAE)

```
일반 Autoencoder: 잠재 표현이 고정된 벡터
VAE: 잠재 표현이 확률 분포 (평균 + 분산)

→ 다양한 추천 생성 가능 (샘플링마다 다른 결과)
→ 과적합에 더 강함
→ 최근 추천 시스템에서 많이 연구됨
```

## Wide & Deep

Google (2016). **암기(memorization)**와 **일반화(generalization)**를 동시에 학습한다.

### 구조

```
Wide 부분 (선형 모델):
  입력: 교차 피처 (예: "유저 성별=남 AND 장르=액션")
  → 자주 함께 나타나는 패턴을 암기
  → 희소하지만 직접적인 규칙 학습

Deep 부분 (심층 신경망):
  입력: 밀집 임베딩 벡터
  → 새로운 피처 조합을 일반화
  → 본 적 없는 조합도 추론

         Wide (선형) ──────────────┐
                                   │
         Deep (신경망) ────────────┤
                                   │
                            [ 합산/concat ]
                                  |
                             예측 점수
```

### 왜 둘 다 필요한가?

```
Wide만: "남성 + 액션 = 좋아함"은 잘 맞추지만
        새로운 조합에 대응 못함

Deep만: 일반화는 잘 하지만
        명확한 규칙을 놓칠 수 있음

둘 다: 명확한 패턴은 Wide가, 새로운 패턴은 Deep이 처리
```

## DeepFM

Wide & Deep의 개선. Wide 부분을 **FM(Factorization Machine)**으로 대체한다.

### FM (Factorization Machine) 간단 정리

```
선형 모델:    y = w0 + Σ wi×xi
             → 피처 간 상호작용 없음

FM:          y = w0 + Σ wi×xi + Σ Σ <vi, vj>×xi×xj
             → 모든 피처 쌍의 상호작용을 잠재 벡터 내적으로 학습
             → 희소 데이터에서도 상호작용 학습 가능
```

### DeepFM 구조

```
입력 피처 ──┬── FM 부분: 피처 상호작용 (2차) ───┐
            │                                    │
            └── DNN 부분: 고차 상호작용 ──────────┤
                                                  │
                                           [ 합산 ]
                                               |
                                          예측 점수

→ Wide & Deep과 달리 피처 엔지니어링 불필요
→ FM이 자동으로 피처 교차를 학습
```

## 딥러닝 추천의 장단점

### 장점

```
1. 비선형 관계 학습 — MF가 못 잡는 복잡한 패턴 포착
2. 다양한 입력 처리 — 텍스트, 이미지, 오디오 등을 함께 활용
3. 유연한 구조 — 문제에 맞게 아키텍처 설계 가능
4. End-to-End 학습 — 피처 추출부터 예측까지 한번에
```

### 단점

```
1. 데이터 요구량 — 적은 데이터에서는 MF보다 못할 수 있음
2. 계산 비용 — 학습/추론에 GPU 필요
3. 해석 어려움 — 왜 이걸 추천했는지 설명하기 어려움
4. 과적합 위험 — 파라미터가 많아 과적합되기 쉬움
5. 하이퍼파라미터 — 층 수, 차원, 학습률 등 조정할 것이 많음
```

### 언제 딥러닝을 쓰는가?

```
딥러닝이 유리:
  - 데이터가 매우 많을 때 (수억 건 이상)
  - 다양한 피처를 활용할 때 (텍스트, 이미지 등)
  - 복잡한 패턴이 있을 때

MF가 유리:
  - 데이터가 적을 때
  - 순수 협업 필터링만 할 때
  - 빠른 학습/추론이 필요할 때
  - 해석 가능성이 중요할 때
```

## 정리

```
딥러닝 추천 = 신경망으로 사용자-아이템 관계의 비선형 패턴을 학습

핵심 모델:
  NCF/NeuMF: MF의 내적을 신경망으로 대체
  Autoencoder: 상호작용 벡터 압축/복원으로 빈 칸 예측
  Wide & Deep: 암기(Wide) + 일반화(Deep) 동시 학습
  DeepFM: FM + DNN으로 피처 상호작용 자동 학습

핵심 구성요소:
  임베딩: ID → 밀집 벡터 변환, 딥러닝 추천의 기반

실무:
  대규모 서비스에서는 딥러닝이 표준
  소규모에서는 MF가 더 효율적일 수 있음

→ 다음 장에서 "순서"를 고려하는 시퀀스 모델을 다룬다
```
