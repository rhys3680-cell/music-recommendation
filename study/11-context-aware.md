# 11. 컨텍스트 인식 추천 (Context-Aware Recommendation)

## 왜 컨텍스트인가?

같은 사용자라도 **상황에 따라 원하는 것이 다르다**.

```
기존 추천:
  f(유저, 아이템) = 선호도
  → "이 유저는 이 아이템을 좋아하는가?"

컨텍스트 인식 추천:
  f(유저, 아이템, 컨텍스트) = 선호도
  → "이 유저가 이 상황에서 이 아이템을 좋아하는가?"
```

```
예시:
  같은 유저가:
    평일 아침 출근길 → 뉴스 팟캐스트
    금요일 저녁      → 파티 음악
    일요일 오후      → 잔잔한 재즈
    비 오는 날       → 감성 발라드

  → 유저-아이템 관계만으로는 이걸 구분할 수 없다
```

## 컨텍스트의 종류

### 1. 시간 컨텍스트

```
시간대:    아침 / 점심 / 저녁 / 밤
요일:      평일 / 주말
계절:      봄 / 여름 / 가을 / 겨울
공휴일:    연휴 / 일반일
```

### 2. 장소 컨텍스트

```
위치:      집 / 사무실 / 이동 중 / 헬스장
국가/도시: 지역별 선호 차이
날씨:      맑음 / 비 / 눈
```

### 3. 디바이스 컨텍스트

```
기기:      모바일 / PC / 태블릿 / 스마트TV
네트워크:  WiFi / 셀룰러 (동영상 품질 결정)
```

### 4. 사회적 컨텍스트

```
혼자 / 친구와 / 가족과 / 연인과
→ 영화 선택이 완전히 달라짐
```

### 5. 사용자 상태

```
기분:      즐거움 / 우울 / 에너지 넘침 / 차분
활동:      운동 중 / 공부 중 / 운전 중 / 수면 전
```

## 컨텍스트 반영 방법

### 방법 1: Contextual Pre-filtering

추천 **전에** 컨텍스트로 데이터를 필터링한다.

```
전체 데이터:
  (유저A, 영화1, 평점5, 주말저녁)
  (유저A, 영화2, 평점3, 평일아침)
  (유저A, 영화3, 평점4, 주말저녁)
  (유저B, 영화1, 평점4, 주말저녁)
  ...

현재 컨텍스트: 주말저녁

필터링 후 데이터:
  (유저A, 영화1, 평점5)  ← 주말저녁 데이터만
  (유저A, 영화3, 평점4)
  (유저B, 영화1, 평점4)

→ 필터링된 데이터로 기존 CF/MF 적용
```

```
장점: 기존 알고리즘을 그대로 사용 가능
단점: 데이터가 쪼개져서 희소성 악화
      컨텍스트 세분화할수록 데이터 부족
```

### 방법 2: Contextual Post-filtering

기존 방식으로 추천한 **후에** 컨텍스트로 조정한다.

```
1단계: 기존 CF로 추천 목록 생성
  [영화A(4.5), 영화B(4.3), 영화C(4.1), 영화D(4.0), 영화E(3.9)]

2단계: 현재 컨텍스트 "평일 점심시간"으로 필터/재정렬
  영화A(2시간) → 제외 (점심시간에 너무 김)
  영화C(90분) → 제외
  영화B(30분 에피소드) → 유지, 1위로
  영화D(25분) → 유지
  영화E(20분) → 유지

최종: [영화B, 영화D, 영화E]
```

```
장점: 기존 시스템에 쉽게 추가 가능
단점: 컨텍스트가 추천 자체에 영향을 주지 못함
      필터링으로 좋은 추천이 제거될 수 있음
```

### 방법 3: Contextual Modeling

컨텍스트를 모델 **내부에 직접 통합**한다.

```
기존 MF:
  R̂(u, i) = P_u · Q_i

컨텍스트 포함:
  R̂(u, i, c) = f(P_u, Q_i, C_c)

→ 컨텍스트도 잠재 벡터로 표현하여 함께 학습
```

이 방식이 가장 강력하다. 대표적인 모델이 **Factorization Machines**이다.

## Factorization Machines (FM)

Rendle (2010). 모든 피처 간의 상호작용을 **효율적으로** 학습하는 범용 모델이다.

### 핵심 아이디어

```
선형 모델:
  y = w0 + w1×x1 + w2×x2 + ... + wn×xn
  → 피처 간 상호작용 없음

다항식 모델:
  y = w0 + Σ wi×xi + Σ Σ wij×xi×xj
  → 모든 피처 쌍의 가중치 wij를 학습해야 함
  → 희소 데이터에서 wij 학습 불가 (xi, xj가 동시에 관측된 데이터 부족)

FM:
  y = w0 + Σ wi×xi + Σ Σ <vi, vj>×xi×xj
  → wij 대신 잠재 벡터 vi, vj의 내적으로 대체
  → 직접 관측 안 된 쌍도 잠재 벡터를 통해 추론 가능
```

### 추천에서의 FM 입력

```
하나의 학습 샘플:

유저(one-hot) | 아이템(one-hot) | 시간대  | 요일  | 디바이스 | 평점
[0,1,0,...,0] | [0,0,1,...,0]   | [0,1,0] | [1,0] | [0,1]   |  4

x = [0,1,0,...,0,  0,0,1,...,0,  0,1,0,  1,0,  0,1]
y = 4

→ 유저, 아이템, 컨텍스트를 모두 하나의 피처 벡터로 합침
→ FM이 "유저 × 시간대", "아이템 × 요일" 등 모든 상호작용 학습
```

### FM의 장점

```
1. 범용성: 유저, 아이템, 컨텍스트를 동일하게 취급
   → MF, SVD++, 컨텍스트 모델이 모두 FM의 특수한 경우

2. 희소 데이터에 강함: 잠재 벡터로 간접 학습
   → "유저A × 비오는날"을 직접 안 봐도
   → "유저A의 벡터"와 "비오는날의 벡터"로 추론

3. 선형 시간 복잡도: O(kn) (k=잠재 차원, n=피처 수)
   → 수학적 트릭으로 효율적 계산
```

### 계산 트릭

```
원래: Σ Σ <vi, vj>×xi×xj → O(k×n²)

변환:
Σ Σ <vi,vj>×xi×xj = 1/2 × Σ( (Σ vi,f×xi)² - Σ (vi,f×xi)² )
                          f=1   i=1            i=1

→ O(k×n)으로 줄어듦!
```

## FFM (Field-aware Factorization Machines)

FM의 확장. 피처가 어떤 **필드(그룹)**에 속하는지 고려한다.

```
FM:
  유저A의 잠재 벡터는 하나 → 모든 피처와 같은 벡터로 상호작용

FFM:
  유저A가 아이템과 상호작용할 때의 벡터 ≠ 시간대와 상호작용할 때의 벡터

  v_유저A_for_아이템필드 = [0.2, 0.5, 0.3]
  v_유저A_for_시간필드   = [0.8, 0.1, 0.6]
  v_유저A_for_디바이스필드 = [0.4, 0.7, 0.2]

→ 더 세밀한 상호작용 모델링
→ 단, 파라미터 수 증가 (벡터 수 = 피처 수 × 필드 수)
```

## 딥러닝 + 컨텍스트

### Deep FM (복습 + 컨텍스트 관점)

```
입력 피처:
  유저 임베딩 + 아이템 임베딩 + 컨텍스트 임베딩

  유저 ID    → [0.2, 1.5, ...]
  아이템 ID  → [0.8, 0.3, ...]
  시간대     → [0.5, 0.9, ...]    ← 컨텍스트
  요일       → [0.3, 0.7, ...]    ← 컨텍스트
  디바이스   → [0.1, 0.4, ...]    ← 컨텍스트

→ FM이 2차 상호작용 학습
→ DNN이 고차 상호작용 학습
→ 컨텍스트가 자연스럽게 통합됨
```

### xDeepFM, DCN (Deep & Cross Network)

```
피처 교차를 더 명시적으로 학습하는 발전된 모델들

DCN: Cross Network로 피처 교차를 효율적으로 계산
xDeepFM: Compressed Interaction Network (CIN)으로 벡터 수준 교차

→ 컨텍스트 피처 간의 복잡한 상호작용을 더 잘 포착
```

## 정리

```
컨텍스트 인식 추천 = 사용자의 현재 상황을 반영한 추천

컨텍스트 종류:
  시간, 장소, 디바이스, 사회적 상황, 사용자 상태

반영 방법:
  Pre-filtering:  컨텍스트로 데이터 필터 후 추천 (간단하지만 희소)
  Post-filtering: 추천 후 컨텍스트로 조정 (기존 시스템에 쉽게 추가)
  Modeling:       모델 내부에 통합 (가장 강력)

핵심 모델:
  FM: 모든 피처 상호작용을 잠재 벡터로 효율적 학습
  FFM: 필드별 다른 잠재 벡터 사용
  DeepFM/DCN: 딥러닝 + 피처 교차의 결합

→ Phase 3 심화가 끝났다.
→ Phase 4에서는 Cold Start, 확장성, 다양성 등 실전 문제를 다룬다.
```
