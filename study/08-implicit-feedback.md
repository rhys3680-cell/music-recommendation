# 8. 암시적 피드백 처리 (Implicit Feedback)

## 왜 별도로 다루는가?

앞에서 배운 알고리즘들은 대부분 **명시적 피드백(평점)**을 전제로 설계되었다. 하지만 실제 서비스에서는 대부분 암시적 피드백이다.

```
명시적 피드백:
  유저A가 영화X에 별점 4점 → "이 유저는 이 영화를 좋아한다"

암시적 피드백:
  유저A가 영화X를 클릭함 → 좋아하는 건가? 실수 클릭인가?
  유저A가 영화Y를 안 봄 → 싫어하는 건가? 모르는 건가?
```

### 암시적 피드백의 특성

```
1. 부정 신호가 없다
   → 안 본 것 = 싫어하는 것? 모르는 것? 구분 불가

2. 노이즈가 많다
   → 클릭했지만 바로 이탈, 실수 구매, 선물 구매 등

3. 값의 의미가 다르다
   → 평점 5 = 확실히 좋아함
   → 재생 5회 = 좋아하는 정도를 정확히 모름 (배경 재생일 수도)

4. 데이터가 풍부하다
   → 모든 사용자가 생성하므로 양이 압도적
```

---

## 기존 MF를 그대로 쓰면 안 되는 이유

```
명시적 피드백 MF:
  관측된 (유저, 아이템, 평점) → 평점 예측 최소화
  관측 안 된 것 = 아직 모르는 것 → 무시

암시적 피드백에 같은 방식 적용:
  관측된 (유저, 아이템, 클릭수) → 클릭수 예측?
  관측 안 된 것 = 관심 없는 것? → 무시하면 안 됨!

문제:
  관측 안 된 것을 무시하면
  → 모델이 관측된 것만 높이려 하고
  → 관측 안 된 것에 대한 예측이 엉망이 됨
```

## Weighted Matrix Factorization (WMF)

Hu et al. (2008)이 제안한 암시적 피드백 전용 MF이다.

### 핵심 아이디어

두 가지 개념을 분리한다:

```
1. 선호도 (Preference): 좋아하는가 아닌가 (이진값)
   p_ui = 1  (관측됨, 즉 상호작용 있음)
   p_ui = 0  (관측 안 됨)

2. 신뢰도 (Confidence): 그 선호를 얼마나 믿을 수 있는가
   c_ui = 1 + α × r_ui

   r_ui = 원본 값 (재생 횟수, 클릭 수 등)
   α = 신뢰도 증가율 (하이퍼파라미터)
```

### 왜 분리하는가?

```
유저A: 곡X 재생 50회 → p=1 (좋아함), c=1+α×50 (매우 확신)
유저A: 곡Y 재생 1회  → p=1 (좋아함?), c=1+α×1  (확신 낮음)
유저A: 곡Z 재생 0회  → p=0 (안 좋아함?), c=1     (확신 매우 낮음)

→ 0회 재생이라고 확실히 싫어하는 건 아니므로 신뢰도를 낮게 설정
→ 50회 재생은 확실히 좋아하므로 신뢰도를 높게 설정
```

### 목적 함수

```
min Σ c_ui × (p_ui - P_u · Q_i)² + λ(|P|² + |Q|²)
   u,i

→ 모든 (u, i) 쌍에 대해 계산 (관측 안 된 것도 포함!)
→ 신뢰도 c_ui로 각 쌍의 중요도를 조절
→ 재생 많이 한 것은 예측 오차에 큰 페널티
→ 재생 0회인 것은 오차에 작은 페널티
```

### 기존 MF와의 차이

| 구분 | 기존 MF (명시적) | WMF (암시적) |
|------|-----------------|-------------|
| 학습 대상 | 관측된 쌍만 | **모든 쌍** |
| 예측 목표 | 평점 값 | 선호 여부 (0/1) |
| 가중치 | 동일 | 신뢰도 기반 차등 |
| 계산량 | 관측 수에 비례 | 유저×아이템 전체 |

### 계산량 문제와 해결

```
모든 (u, i) 쌍 = 100만 유저 × 10만 아이템 = 1,000억 쌍
→ SGD로는 불가능

해결: ALS 사용
→ P 고정 시 Q의 최적해를 행렬 연산으로 한번에 계산
→ 효율적인 수학적 트릭으로 O(k²(m+n) + k²|관측|) 수준으로 줄임
```

## BPR (Bayesian Personalized Ranking)

Rendle et al. (2009)이 제안. WMF와 다른 접근으로 암시적 피드백을 처리한다.

### 핵심 아이디어

```
평점을 예측하는 게 아니라, 아이템 간의 "순서"를 학습한다.

"유저A가 곡X를 들었고 곡Y를 안 들었다"
→ 유저A는 곡Y보다 곡X를 선호한다
→ score(A, X) > score(A, Y)가 되도록 학습
```

### 학습 데이터 구성

```
각 사용자에 대해 (유저, 긍정 아이템, 부정 아이템) 삼중쌍을 만든다.

유저A 이력: 곡X(재생), 곡Y(재생), 곡Z(미재생), 곡W(미재생)

삼중쌍:
  (A, X, Z) → X를 Z보다 선호  
  (A, X, W) → X를 W보다 선호
  (A, Y, Z) → Y를 Z보다 선호
  (A, Y, W) → Y를 W보다 선호
```

### 목적 함수

```
max Σ ln σ(score(u,i) - score(u,j)) - λ|Θ|²

(u, i, j) = (유저, 긍정 아이템, 부정 아이템)
σ = 시그모이드 함수
Θ = 모델 파라미터 (P, Q)

→ 긍정 아이템 점수가 부정 아이템 점수보다 높아지도록 학습
```

### 학습 과정

```
1. 랜덤으로 삼중쌍 (u, i, j) 샘플링
2. 예측 차이 계산: x_uij = score(u,i) - score(u,j)
3. 그래디언트 계산하여 파라미터 업데이트
4. 반복

→ SGD로 효율적 학습 가능 (관측된 쌍 기반 샘플링)
```

### WMF vs BPR

| 구분 | WMF | BPR |
|------|-----|-----|
| 관점 | 선호도 예측 (회귀) | 순위 학습 (랭킹) |
| 학습 대상 | 모든 (u, i) 쌍 | 샘플링된 삼중쌍 |
| 최적화 | ALS | SGD |
| 계산 효율 | 행렬 연산 필요 | 샘플링 기반, 가벼움 |
| 평가 지표와의 관계 | RMSE 계열 | **AUC, NDCG 계열** |
| 실무 적합성 | 배치 추천 | **랭킹 기반 추천** |

## 부정 샘플링 (Negative Sampling)

BPR이나 다른 암시적 피드백 모델에서 **부정 샘플을 어떻게 뽑느냐**가 성능에 큰 영향을 미친다.

### 균일 샘플링 (Uniform)

```
관측 안 된 아이템 중 랜덤으로 부정 샘플 선택

장점: 간단
단점: 대부분 너무 쉬운 부정 샘플 (인기 없는 아이템)
     → 모델이 쉬운 것만 학습하고 어려운 구분을 못함
```

### 인기도 기반 샘플링

```
인기 아이템을 부정 샘플로 더 자주 선택

P(j) ∝ popularity(j)^0.75

→ "인기 있는데 이 유저가 안 본 것" = 더 의미 있는 부정 신호
→ Word2Vec의 네거티브 샘플링에서 영감
```

### Hard Negative Sampling

```
현재 모델이 높은 점수를 주지만 실제로 관측 안 된 아이템을 부정 샘플로 선택

→ 모델이 헷갈려하는 것을 집중 학습
→ 학습 효율이 높지만 불안정할 수 있음
```

## 실무에서의 암시적 피드백 처리

### 시그널 조합

하나의 암시적 시그널만 쓰지 않고 **여러 시그널을 조합**한다.

```
음악 서비스 예시:
  재생 완료:     가중치 1.0
  재생 70% 이상: 가중치 0.7
  재생 30% 미만: 가중치 0.1 (스킵으로 간주)
  좋아요:        가중치 2.0
  플레이리스트 추가: 가중치 1.5
  곡 스킵:       가중치 -0.5 (부정 신호)

최종 선호 점수 = Σ(시그널 × 가중치)
```

### 시간 감쇠 (Time Decay)

```
오래된 행동일수록 가중치를 줄인다.

weight = base_weight × decay^(현재시간 - 행동시간)

→ 1년 전 들은 곡보다 어제 들은 곡이 더 중요
→ 사용자 취향 변화를 반영
```

## 정리

```
암시적 피드백 = 사용자 행동에서 추론한 선호 (클릭, 재생, 구매 등)

핵심 문제:
  "안 본 것 = 싫어하는 것"이 아니다
  → 부정 신호 부재, 노이즈, 신뢰도 차이

해결 방법:
  WMF: 선호도(0/1) + 신뢰도(행동 강도)를 분리
       모든 쌍을 학습하되 신뢰도로 가중치 조절

  BPR: 아이템 간 순위를 학습
       "X를 Y보다 선호한다"는 쌍 비교
       랭킹 지표와 직접 연결

  부정 샘플링: 어떤 부정 샘플을 쓰느냐가 성능 좌우

실무:
  여러 시그널 조합 + 시간 감쇠로 선호도 점수 구성
  → 이 점수를 모델 입력으로 사용
```
