# 14. 공정성과 다양성 (Fairness & Diversity)

## 왜 정확도만으로는 부족한가?

```
정확도만 추구하면 생기는 문제:

유저A가 액션 영화를 많이 봤다
→ 모델: 액션만 추천하면 정확도 높음!
→ 유저A는 영원히 액션만 보게 됨
→ 실제로는 로맨스도 좋아할 수 있었는데 기회를 잃음

인기 곡이 추천을 독식
→ 모델: 인기 곡 추천하면 평균 정확도 높음!
→ 신인 아티스트는 노출 기회 없음
→ 서비스 생태계가 죽음
```

좋은 추천 시스템은 **정확하면서도 공정하고 다양해야** 한다.

## 필터 버블 (Filter Bubble)

### 문제

```
추천 → 소비 → 데이터 → 추천 강화 → 더 좁은 소비 → ...

피드백 루프:
  유저가 액션 봄 → 액션 추천 → 액션 더 봄 → 액션만 추천
  → 점점 좁은 범위에 갇힘 = 필터 버블

정보 편향:
  뉴스: 특정 정치 성향의 기사만 보게 됨
  음악: 한 장르에만 갇힘
  쇼핑: 비슷한 가격대/브랜드만 노출
```

### 왜 위험한가?

```
사용자 관점:
  - 새로운 발견의 기회 상실
  - 취향이 고착화
  - 장기적으로 서비스에 대한 피로감

서비스 관점:
  - 사용자 이탈 (지루함)
  - 롱테일 아이템 사장
  - 생태계 불균형 (인기 쏠림)

사회적 관점:
  - 정보 편향 심화
  - 극단화 (뉴스, 영상)
```

## 다양성 (Diversity)

### 다양성의 정의

```
개인 내 다양성 (Intra-list Diversity):
  한 사용자의 추천 목록 안에서 아이템들이 얼마나 다양한가
  → [액션, 액션, 액션, 액션, 액션] = 다양성 낮음
  → [액션, 로맨스, 다큐, 코미디, SF] = 다양성 높음

전체 다양성 (Aggregate Diversity):
  모든 사용자의 추천을 합쳤을 때 얼마나 다양한 아이템이 추천되는가
  → 100명 모두에게 같은 10개 추천 = 다양성 낮음
  → 100명에게 각각 다른 아이템 추천 = 다양성 높음
```

### 측정 지표

#### ILD (Intra-List Distance)

```
추천 목록 내 모든 아이템 쌍의 평균 거리

ILD = (2 / (n×(n-1))) × ΣΣ dist(i, j)

예시:
  추천 목록: [액션A, 액션B, 로맨스C]
  
  dist(액션A, 액션B) = 0.1  (비슷)
  dist(액션A, 로맨스C) = 0.8  (다름)
  dist(액션B, 로맨스C) = 0.7  (다름)
  
  ILD = (2/6) × (0.1 + 0.8 + 0.7) = 0.53
```

#### Coverage (커버리지)

```
Coverage = 추천된 고유 아이템 수 / 전체 아이템 수

전체 아이템 10,000개인데 추천에 등장한 건 500개
→ Coverage = 5%
→ 95%의 아이템은 한 번도 추천되지 않음
```

#### Gini Index (지니 계수)

```
추천 횟수의 불평등도 측정

0에 가까움 → 모든 아이템이 균등하게 추천됨
1에 가까움 → 소수 아이템이 추천을 독점

실제 서비스: 0.8~0.95 수준 (극심한 불균형)
```

### 다양성 확보 방법

#### 1. MMR (Maximal Marginal Relevance)

관련성과 다양성을 **동시에 최적화**하는 탐욕 알고리즘이다.

```
하나씩 추천 목록에 추가할 때:

score(i) = λ × relevance(i) - (1-λ) × max_sim(i, 이미 선택된 아이템)

λ = 0.7이면: 관련성 70%, 다양성 30%

과정:
  1번째: 관련성 가장 높은 아이템 선택
  2번째: 관련성 높으면서 1번째와 다른 아이템 선택
  3번째: 관련성 높으면서 1,2번째와 다른 아이템 선택
  ...
```

```
예시 (λ=0.5):

후보: 액션A(0.9), 액션B(0.85), 로맨스C(0.8), SF D(0.75)

1번째: 액션A 선택 (관련성 최고)

2번째 점수:
  액션B:  0.5×0.85 - 0.5×sim(액션B, 액션A)
        = 0.425 - 0.5×0.9 = -0.025
  로맨스C: 0.5×0.8 - 0.5×sim(로맨스C, 액션A)
         = 0.4 - 0.5×0.2 = 0.3
  SF D:   0.5×0.75 - 0.5×sim(SF D, 액션A)
        = 0.375 - 0.5×0.3 = 0.225

→ 로맨스C 선택 (관련성도 있고 액션A와 다름)
```

#### 2. DPP (Determinantal Point Process)

수학적으로 **다양성을 보장하는 확률 모델**이다.

```
아이디어:
  "비슷한 아이템 쌍이 동시에 선택될 확률을 낮춘다"

행렬 L을 구성:
  L_ij = relevance(i) × sim(i,j) × relevance(j)

이 행렬의 행렬식(determinant)이 큰 부분집합 = 관련성 높고 다양한 집합

→ MMR보다 이론적으로 우수하지만 계산 비용이 높음
→ 근사 알고리즘으로 실무 적용
```

#### 3. 카테고리 제약

단순하지만 효과적인 규칙 기반 방법이다.

```
규칙 예시:
  - 같은 장르 연속 3개 이하
  - 같은 아티스트 2개 이하
  - 최소 3개 장르 포함
  - 신규 아이템 최소 20% 포함

재정렬 단계에서 적용하면 간단하게 다양성 확보
```

## 공정성 (Fairness)

### 공정성의 대상

```
사용자 공정성 (Consumer Fairness):
  "모든 사용자가 공정한 품질의 추천을 받는가?"
  → 소수 그룹 사용자의 추천 품질이 떨어지지 않는가?

공급자 공정성 (Provider Fairness):
  "모든 아이템 제공자가 공정한 노출 기회를 받는가?"
  → 소규모 판매자, 신인 아티스트도 노출 기회가 있는가?

아이템 공정성 (Item Fairness):
  "유사한 품질의 아이템이 유사한 추천 기회를 받는가?"
  → 인기 편향으로 좋은 비인기 아이템이 묻히지 않는가?
```

### 사용자 공정성 문제

```
데이터 불균형:
  활발한 유저 → 데이터 많음 → 추천 정확도 높음
  비활발 유저 → 데이터 적음 → 추천 정확도 낮음
  
  → 비활발 유저일수록 더 나쁜 추천을 받음
  → 이런 유저가 대다수

인구통계 편향:
  학습 데이터에 특정 그룹이 적으면
  → 해당 그룹에 대한 추천 성능이 떨어짐
```

### 공급자 공정성 문제

```
인기 편향 (Popularity Bias):
  인기 아이템 → 많이 추천됨 → 더 많이 소비 → 더 인기
  비인기 아이템 → 추천 안 됨 → 소비 안 됨 → 더 비인기

  → 부익부 빈익빈 (Matthew Effect)

수치로 보면:
  상위 1% 아이템이 전체 추천의 60%를 차지
  하위 50% 아이템은 전체 추천의 2%도 안 됨
```

### 공정성 확보 방법

#### 1. 인기도 보정

```
역인기도 가중치 (Inverse Popularity Weighting):
  score_adjusted = score × (1 / popularity)^α

  인기 아이템: score는 높지만 가중치가 낮아짐
  비인기 아이템: 상대적으로 가중치가 올라감

  α로 보정 강도 조절 (0이면 보정 없음, 1이면 강한 보정)
```

#### 2. 노출 공정성 제약

```
최적화 문제에 공정성 제약 추가:

max Σ relevance(추천)
subject to:
  exposure(그룹A) / merit(그룹A) ≈ exposure(그룹B) / merit(그룹B)

→ 품질 대비 노출 비율이 그룹 간 균등하도록
```

#### 3. 공정성 인식 학습

```
학습 목적 함수에 공정성 항 추가:

Loss = 추천 손실 + β × 공정성 페널티

공정성 페널티 예시:
  - 그룹 간 추천 정확도 차이
  - 노출 비율의 지니 계수
  - 인기도와 추천 횟수의 상관계수
```

## Exploration vs Exploitation

### 딜레마

```
Exploitation (활용):
  이미 좋다고 알고 있는 것을 추천
  → 단기 만족도 높음
  → 장기적으로 필터 버블

Exploration (탐색):
  잘 모르는 것을 추천하여 정보 수집
  → 단기 만족도 낮을 수 있음
  → 장기적으로 더 좋은 추천 가능
```

### 해결 방법

#### Epsilon-Greedy

```
ε = 0.1 (10%)

90% 확률: 최적 추천 (Exploitation)
10% 확률: 랜덤 추천 (Exploration)

장점: 매우 단순
단점: 탐색이 완전 랜덤 (품질 보장 없음)
```

#### UCB (Upper Confidence Bound)

```
score(i) = 예측값(i) + c × √(ln(전체 노출) / 노출(i))

→ 노출이 적은 아이템일수록 두 번째 항(불확실성 보너스)이 커짐
→ "잘 모르는 아이템은 한번 보여주자"
→ 시도한 후 별로면 자연스럽게 점수 하락
```

#### Thompson Sampling

```
각 아이템의 성과를 확률 분포로 모델링
매 추천 시 분포에서 샘플링하여 점수 결정

→ 불확실성이 큰 아이템은 때때로 높은 점수가 나옴
→ 자연스러운 탐색과 활용의 균형
→ 실무에서 가장 효과적이라는 평가
```

```
Spotify Discover Weekly:
  익숙한 취향 70% + 새로운 발견 30%
  → 매주 다양한 곡을 포함하여 필터 버블 방지

YouTube:
  관련성 높은 영상 + "이런 건 어때요?" 섹션
  → 별도 슬롯으로 탐색 영역 확보

Netflix:
  카테고리 행(row)별로 다른 전략
  → "인기작", "당신을 위한 추천", "새로운 발견" 등 혼합
```

## 정리

```
공정성과 다양성 = 정확도 너머의 추천 품질

필터 버블:
  추천 → 소비 → 데이터 → 추천 강화의 악순환
  → 사용자 경험 저하, 생태계 불균형

다양성 확보:
  MMR: 관련성 + 다양성 동시 최적화 (가장 실용적)
  DPP: 수학적 다양성 보장 (이론적으로 우수)
  카테고리 제약: 규칙 기반 (간단하고 효과적)

공정성 확보:
  인기도 보정, 노출 공정성 제약, 공정성 인식 학습

Explore vs Exploit:
  Epsilon-Greedy: 단순 (일정 비율 랜덤)
  UCB: 불확실성 보너스 (적게 노출된 것 우대)
  Thompson Sampling: 확률적 탐색 (실무 최적)

핵심:
  정확도와 다양성은 트레이드오프가 아니라
  장기적으로는 다양성이 정확도도 높인다
```
