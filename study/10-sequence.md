# 10. 시퀀스/세션 기반 추천 (Sequence & Session-Based Recommendation)

## 왜 순서가 중요한가?

지금까지의 모델은 **순서를 무시**했다.

```
기존 CF/MF:
  유저A = {영화1, 영화2, 영화3}  (집합)
  → 순서 상관없이 동일하게 취급

현실:
  유저A: 로맨스 → 로맨스 → 로맨스 → 액션 → 액션
  → 최근 취향이 액션으로 바뀌고 있다!
  → 기존 모델은 이걸 반영하지 못함
```

```
순서가 중요한 도메인:
  음악:    느린 곡 → 느린 곡 → 갑자기 빠른 곡? (흐름이 중요)
  쇼핑:    검색 → 비교 → 장바구니 → 구매 (구매 여정)
  뉴스:    정치 기사 → 정치 기사 → 다음도 정치? (관심 흐름)
  영상:    시즌1 → 시즌2 → 시즌3 (시청 순서)
```

## 시퀀스 vs 세션

```
시퀀스 기반:
  사용자의 전체 이력을 시간순으로 본다
  [1주 전 본 것, 3일 전 본 것, 어제 본 것, 오늘 본 것]
  → 장기 취향 변화를 포착

세션 기반:
  하나의 방문(세션) 내 행동만 본다
  [이번 접속에서 클릭한 것들]
  → 현재 의도를 포착
  → 로그인 안 한 익명 사용자에게도 적용 가능
```

## Markov Chain 기반

가장 단순한 시퀀스 모델. **직전 아이템만 보고** 다음 아이템을 예측한다.

### 1차 Markov Chain

```
P(다음 아이템 | 현재 아이템)

전이 확률 행렬:
          다음에 A    다음에 B    다음에 C
현재 A      0.1        0.6        0.3
현재 B      0.4        0.1        0.5
현재 C      0.5        0.3        0.2

유저가 지금 A를 봤다면 → B를 추천 (60% 확률)
```

### 한계

```
1차: 바로 직전만 봄 → 너무 근시안적
2차: 직전 2개 봄   → 상태 공간 폭발 (아이템수²)
고차: 직전 n개 봄  → 상태 공간 폭발 (아이템수^n)

→ 긴 이력을 반영하기 어렵다
→ RNN이 이 문제를 해결
```

## GRU4Rec

Hidasi et al. (2016). **GRU(Gated Recurrent Unit)**를 세션 기반 추천에 적용한 첫 모델이다.

### GRU란?

RNN의 변형으로, **장기 의존성 문제를 완화**한 구조이다.

```
일반 RNN:
  h_t = tanh(W × [h_{t-1}, x_t])
  → 긴 시퀀스에서 기울기 소실/폭발 문제

GRU:
  z_t = σ(Wz × [h_{t-1}, x_t])        ← 업데이트 게이트
  r_t = σ(Wr × [h_{t-1}, x_t])        ← 리셋 게이트
  h̃_t = tanh(W × [r_t ⊙ h_{t-1}, x_t])  ← 후보 은닉 상태
  h_t = (1-z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t   ← 최종 은닉 상태

→ 게이트로 "무엇을 기억하고 무엇을 잊을지" 학습
```

### GRU4Rec 구조

```
세션 내 클릭 시퀀스: [아이템A, 아이템B, 아이템C, ?]

아이템A      아이템B      아이템C
  ↓            ↓            ↓
임베딩       임베딩       임베딩
  ↓            ↓            ↓
[GRU] ───→ [GRU] ───→ [GRU]
                          ↓
                     은닉 상태 h₃
                          ↓
                    [출력 Layer]
                          ↓
                 모든 아이템에 대한 점수
                          ↓
                    다음 아이템 예측

→ h₃에 "A→B→C 순서로 본 사용자의 현재 의도"가 압축됨
```

### 학습

```
손실 함수: BPR loss 또는 Cross-Entropy

각 타임스텝에서:
  입력: 현재까지의 시퀀스
  목표: 다음에 클릭한 아이템

[A] → 다음은 B  (학습)
[A, B] → 다음은 C  (학습)
[A, B, C] → 다음은 ?  (예측)
```

## Self-Attention 기반: SASRec

Kang & McAuley (2018). Transformer의 Self-Attention을 추천에 적용했다.

### RNN의 한계

```
RNN/GRU:
  A → B → C → D → E
  
  E를 처리할 때 A의 정보는 여러 단계를 거쳐 전달
  → 긴 시퀀스에서 초기 정보 손실
  → 순차 처리라 병렬화 불가능
```

### Self-Attention이란?

```
시퀀스의 모든 위치가 다른 모든 위치를 직접 참조한다.

[A, B, C, D, E]

E를 예측할 때:
  A와의 관련도: 0.1 (오래됐지만 관련 있을 수 있음)
  B와의 관련도: 0.05
  C와의 관련도: 0.3
  D와의 관련도: 0.55 (가장 관련)

→ 거리에 상관없이 관련 있는 아이템에 직접 집중
→ 병렬 계산 가능
```

### SASRec 구조

```
입력 시퀀스: [아이템1, 아이템2, ..., 아이템n]

아이템 임베딩 + 위치 임베딩 (Position Encoding)
          ↓
  [ Self-Attention Layer 1 ]
          ↓
  [ Feed-Forward Network 1 ]
          ↓
  [ Self-Attention Layer 2 ]
          ↓
  [ Feed-Forward Network 2 ]
          ↓
    최종 표현 벡터
          ↓
  후보 아이템과 내적 → 점수
```

### Attention 계산

```
Q = 쿼리 (현재 위치가 "무엇을 찾고 싶은가")
K = 키   (각 위치가 "나는 이런 정보를 가지고 있다")
V = 값   (각 위치의 실제 정보)

Attention(Q, K, V) = softmax(Q × K^T / √d) × V

√d로 나누는 이유: 차원이 클수록 내적 값이 커지는 것을 보정
```

### Causal Masking (인과적 마스킹)

```
추천에서는 "미래를 보면 안 된다"

마스크 행렬:
       A    B    C    D
  A  [ ✓    ✗    ✗    ✗ ]
  B  [ ✓    ✓    ✗    ✗ ]
  C  [ ✓    ✓    ✓    ✗ ]
  D  [ ✓    ✓    ✓    ✓ ]

→ D를 예측할 때 A,B,C만 참조 가능
→ C를 예측할 때 A,B만 참조 가능
→ 학습 시 한번에 모든 위치의 다음 아이템을 예측 가능 (효율적)
```

## BERT4Rec

Sun et al. (2019). BERT의 **양방향 학습**을 추천에 적용했다.

### SASRec vs BERT4Rec

```
SASRec (단방향):
  [A, B, C, D] → E를 예측
  왼쪽만 참조 가능

BERT4Rec (양방향):
  [A, B, [MASK], D] → C를 예측
  양쪽 모두 참조 가능

→ BERT4Rec는 시퀀스 중간의 빈 칸도 채울 수 있음
→ 더 풍부한 표현 학습
```

### Masked Item Prediction

```
학습:
  원본: [A, B, C, D, E]
  마스킹: [A, [MASK], C, [MASK], E]
  목표: MASK 위치의 아이템 예측 (B, D)

추론:
  입력: [A, B, C, D, [MASK]]
  예측: MASK 위치 = 다음 아이템
```

## GRU4Rec vs SASRec vs BERT4Rec 비교

| 구분 | GRU4Rec | SASRec | BERT4Rec |
|------|---------|--------|----------|
| 기반 | RNN (GRU) | Transformer (단방향) | Transformer (양방향) |
| 참조 범위 | 순차적 전달 | 왼쪽 전체 직접 참조 | 양방향 전체 참조 |
| 병렬화 | 불가능 | **가능** | **가능** |
| 긴 시퀀스 | 약함 | **강함** | **강함** |
| 학습 방식 | 다음 아이템 예측 | 다음 아이템 예측 | 마스크 아이템 예측 |
| 계산 비용 | 낮음 | 보통 | 높음 |
| 성능 | 기본 | 좋음 | **가장 좋음** (보통) |

## 실무 적용 시 고려사항

### 시퀀스 길이

```
너무 짧으면 → 패턴 부족
너무 길면   → 노이즈 포함, 계산 비용 증가

일반적:
  세션 기반: 최근 20~50개
  시퀀스 기반: 최근 50~200개
```

### 시간 정보 활용

```
단순 순서만 보면:
  [곡A, 곡B, 곡C] — 3분 간격인지 3개월 간격인지 모름

시간 임베딩 추가:
  [(곡A, 10:00), (곡B, 10:03), (곡C, 10:05)]
  → 짧은 간격 = 같은 세션, 연속 청취
  → 긴 간격 = 다른 맥락
```

### 장기 + 단기 결합

```
장기 취향: "이 유저는 원래 재즈를 좋아함" (전체 이력)
단기 의도: "지금은 운동 중이라 EDM 듣는 중" (현재 세션)

→ 장기 모델 (MF, 유저 임베딩) + 단기 모델 (시퀀스) 결합
→ 실무에서 많이 쓰이는 패턴
```

## 정리

```
시퀀스/세션 기반 추천 = 사용자 행동의 순서와 흐름을 반영

발전 흐름:
  Markov Chain → GRU4Rec → SASRec → BERT4Rec
  (직전만)       (순차 RNN)  (단방향 Attention)  (양방향 Attention)

핵심 개념:
  Markov: 전이 확률로 다음 아이템 예측 (단순하지만 제한적)
  GRU4Rec: RNN으로 세션 내 의도 학습 (최초의 딥러닝 세션 추천)
  SASRec: Self-Attention으로 긴 의존성 포착 + 병렬화
  BERT4Rec: 양방향 학습으로 더 풍부한 표현

실무 포인트:
  시퀀스 길이 조절, 시간 정보 활용, 장기+단기 취향 결합이 중요

→ 다음 장에서는 시간대, 기분, 활동 등 "맥락"을 반영하는 컨텍스트 인식 추천을 다룬다
```
