# 정규식(re)과 파이썬에서의 사용

## re 모듈

파이썬 표준 라이브러리의 정규식 모듈.

```python
import re
```

## 정규식이란

**문자열의 패턴을 표현하는 언어.**

```
"2026. 4. 11. 오후 6:00:59 KST"
→ 이게 "날짜 형식"이라는 패턴을 따른다는 걸 표현

"ROCKSTAR"
→ 단순 문자열

정규식은 "이런 규칙을 만족하는 문자열"을 찾거나 매칭할 때 쓴다.
```

## 기본 사용

### 패턴 컴파일

```python
pattern = re.compile(r"\d+")
```

- `re.compile()` = 패턴을 미리 해석해서 객체로 반환
- 여러 번 쓸 때 한 번만 컴파일하면 효율적
- 모듈 레벨 상수로 두는 게 관례

### 매칭 메서드

```python
pattern.match(text)     # 문자열 **시작**부터 매칭
pattern.search(text)    # 문자열 **아무 위치**에서 매칭
pattern.findall(text)   # 매칭되는 **모든** 부분 리스트
pattern.sub(repl, text) # 매칭 부분을 repl로 치환
```

### 결과: Match 객체

```python
m = pattern.match(text)
if m:
    m.group(0)       # 매칭된 전체 문자열
    m.group(1)       # 첫 번째 캡처 그룹
    m.groups()       # 모든 캡처 그룹 튜플
    m.start(), m.end()  # 매칭 위치
else:
    # None 반환 (매칭 실패)
```

## raw string (r"...")

```python
r"\d+"    # raw string, 백슬래시 그대로
"\\d+"    # 일반 string, 같은 의미지만 백슬래시를 이스케이프해야 함
```

- 정규식은 백슬래시(`\`)를 많이 쓴다
- raw string을 쓰면 이스케이프 지옥을 피할 수 있다
- **정규식은 항상 `r"..."`로 쓰는 게 관례**

## 주요 메타 문자

### 문자 클래스

```
.        임의의 한 문자 (개행 제외)
\d       숫자 [0-9]
\D       숫자가 아닌 것
\w       단어 문자 [a-zA-Z0-9_]
\W       단어 문자가 아닌 것
\s       공백 (스페이스, 탭, 개행 등)
\S       공백이 아닌 것
[abc]    a, b, c 중 하나
[^abc]   a, b, c가 아닌 것
[a-z]    a~z 범위
```

### 반복

```
*        0회 이상
+        1회 이상
?        0 또는 1회
{n}      정확히 n회
{n,m}    n~m회
{n,}     n회 이상
```

### 위치

```
^        문자열 시작
$        문자열 끝
\b       단어 경계
```

### 그룹

```
(...)    캡처 그룹
(?:...)  비캡처 그룹 (매칭만, 결과에 안 담김)
a|b      a 또는 b
```

## 실전 예: 날짜 패턴

```python
DATE_PATTERN = re.compile(r"^\d{4}\.\s*\d{1,2}\.\s*\d{1,2}\..*KST$")
```

### 패턴 분해

```
^           문자열 시작
\d{4}       숫자 4자리           (예: "2026")
\.          점 하나 (정규식에서 .은 특수문자라 이스케이프)
\s*         공백 0개 이상
\d{1,2}     숫자 1~2자리         (예: "4" 또는 "12")
\.          점
\s*         공백 0개 이상
\d{1,2}     숫자 1~2자리         (예: "11")
\.          점
.*          임의 문자 0개 이상   (오전/오후, 시분초 부분)
KST$        "KST"로 끝남
```

### 왜 이렇게 썼나

처음엔 `"KST" in text`로 단순 포함 검사를 했는데,
제목에 `"ROCKSTAR"`가 들어가면 그것도 매칭되어 버렸다.

```
"ROCKSTAR"에 "KST"가 포함됨
→ text.endswith("KST")로도 방지는 가능하지만 확률적 가정
→ 날짜 패턴 자체를 규정하면 구조적으로 안전
```

## 자주 쓰는 이스케이프 주의

```
.        → \.         리터럴 점
(        → \(         리터럴 괄호
?        → \?         리터럴 물음표
|        → \|         리터럴 파이프
*, +, {  등도 마찬가지
```

`[]` 안에서는 대부분의 특수문자가 리터럴로 취급되지만, `]`, `\`, `^` 는 여전히 특수 취급.

## pandas에서의 정규식

```python
df["title"].str.contains("pattern", regex=True)   # 정규식
df["title"].str.contains("pattern", regex=False)  # 리터럴 치환
df["title"].str.replace(".", "-", regex=False)    # 점을 리터럴로
```

pandas의 `.str.contains`, `.str.replace`는 **기본이 정규식 모드**다.
리터럴 치환이 필요하면 `regex=False` 명시.

## 정리

```
re 모듈 = 파이썬의 정규식 라이브러리

핵심:
  - re.compile(r"...") 로 패턴 객체 생성
  - pattern.match(text) → 시작부터
  - pattern.search(text) → 어디서든
  - raw string(r"...") 관례

정규식:
  \d, \w, \s 등 문자 클래스
  *, +, ?, {n,m} 반복
  ^, $ 위치
  (...) 캡처, (?:...) 비캡처, | OR

실전 교훈:
  "포함" 검사보다 "패턴 매칭"이 더 안전하다
  (ROCKSTAR 사건)
```
