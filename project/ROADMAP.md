# 프로젝트 로드맵

## 목표

내 유튜브 시청 데이터를 기반으로 음악 추천 시스템을 만든다.
- 내 성향 기반 노래 추천
- 노래 ↔ 콘텐츠(영화, 드라마 등) 교차 연결
- GPU 없이 실험 가능한 범위

## Phase 1. 데이터 확보 및 탐색

```
목표: 내 데이터를 확보하고, 어떤 패턴이 있는지 파악한다

1-1. Google Takeout 데이터 파싱
     - 유튜브 시청 기록 파싱
     - 음악 영상만 필터링 (뮤직비디오, 라이브, OST 등)

1-2. 데이터 정제
     - 중복 제거, 결측치 처리
     - 곡명, 아티스트, 영상 유형 분류

1-3. EDA (탐색적 데이터 분석)
     - 가장 많이 들은 아티스트/곡
     - 시간대별 청취 패턴
     - 장르/분위기 분포
     - 소비 추이 변화
```

## Phase 2. 콘텐츠 기반 추천

```
목표: 곡의 속성을 기반으로 유사한 곡을 추천한다

2-1. 곡 메타데이터 수집
     - YouTube Data API로 영상 정보 수집
     - Spotify API로 오디오 특성 수집 (BPM, 에너지, 분위기 등)
     - 장르, 태그, 아티스트 정보

2-2. 아이템 프로필 구축
     - 메타데이터를 벡터로 변환
     - TF-IDF (태그, 설명 기반) 또는 오디오 특성 벡터

2-3. 유사도 기반 추천
     - 코사인 유사도로 유사 곡 검색
     - 내 상위 곡 기반 추천 결과 확인
     - 결과 분석 및 튜닝
```

## Phase 3. 협업 필터링 실험

```
목표: 다른 사용자들의 패턴을 활용하여 추천한다

3-1. 공개 데이터셋 준비
     - Million Song Dataset, Last.fm, 또는 Spotify 공개 데이터
     - 데이터 전처리 및 탐색

3-2. MF 모델 구현
     - SVD/ALS 기반 Matrix Factorization (CPU 가능)
     - 오프라인 평가 (NDCG, Recall@K)

3-3. 내 데이터와 결합
     - 내 청취 이력을 공개 데이터에 매핑
     - CF 기반 추천 결과 확인
```

## Phase 4. 콘텐츠 연결

```
목표: 노래 ↔ 영화/드라마 교차 추천을 구현한다

4-1. OST ↔ 콘텐츠 매핑 데이터 구축
     - OST와 원작 콘텐츠 연결 정보 수집
     - 영화/드라마 메타데이터 (장르, 분위기 등)

4-2. 교차 추천 로직
     - 곡 → 관련 영화/드라마 추천
     - 영화/드라마 → OST 추천
     - 분위기/장르 기반 간접 연결
```

## Phase 5. 통합 및 개선

```
목표: 여러 방법을 결합하고 사용할 수 있는 형태로 만든다

5-1. 하이브리드 추천
     - 콘텐츠 기반 + CF 결합
     - 가중치 조정 및 평가

5-2. 결과 평가
     - 추천 결과 직접 확인 (내가 좋아할 만한 곡인가?)
     - 다양성, 참신성 체크

5-3. 인터페이스
     - CLI 또는 노트북 대시보드
     - 곡/영화 입력 → 추천 결과 출력
```
